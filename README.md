# Publish Precheck｜发布前审

> 发抖音、小红书、微信视频号之前，先让 AI 帮你审一遍：能不能发、哪句有问题、怎么改——并且把你踩过的坑记住，越用越准。

国内平台不会在发布前告诉你哪里会出问题，等它提醒你的时候已经是违规通知了。这个 Skill 把审核前置：基于 30+ 份官方规则文件（广告法、网信办专项、平台规则中心/社区公约/运营规范）提炼的规则库，结合语义判定，在发布前给你明确的结论和可直接替换的改稿。

做内容的都知道那个死循环：选题平淡没流量，选题劲爆就违规。但平台拦的大多是**表达方式**，不是话题本身——同一个劲爆选题，几乎总有合规的爆款在正常拿流量。这个工具帮你找到"同样劲爆但不违规"的说法，而且你每次实测出"这样说被拦、那样说就过"的经验，都会沉淀成你自己的安全表达库，越用越懂你的账号。

## 它和"违禁词表"有什么不同

- **词面命中不等于违规**。"钱""第一"这类词是否有问题取决于语境和商业属性——本 Skill 用"词面预检定位 + 语义判定下结论"两层协作，判定尺度用 38 篇平台实际放行的真实口播稿校准过，不做"见词就杀"的误报机器；
- **每条规则可追溯**。所有规则附官方依据原文引文、出处链接和核验日期，不是"听说不能发"；
- **帮你辟谣**。"赚钱要写成赚米"这类民间谣言，平台官方早已澄清——遇到会主动告诉你不用改；
- **改稿保留你的风格**。修复优先级：补披露 → 补证据 → 收缩主张 → 最小改写 → 删除。绝不提供谐音、拆字等对抗写法（那是在制造新风险）；
- **越用越准**。你的行业敏感词、误报白名单、被罚案例都会沉淀在 `data/` 目录（纯文本，你随时能看能改），之后每次审核自动生效。

## 装好后的第一句话

不知道从哪开始？下面任选一句，复制发给你的 AI 就能用：

```text
用 yuwen-publish-precheck 审一下这篇口播稿能不能发抖音：（贴上稿子）
```

```text
我这条视频被限流了，帮我复盘一下为什么
```

```text
以后帮我盯着"对赌承诺"这个词，同行有人因为它被举报过
```

```text
把我整理的违禁词表导进来（发给它文件）
```

```text
你能帮我做什么？
```

首次使用它会花一分钟引导你建档（常发平台、行业、商业模式），之后审核不再重复问。其他能力（改成能发的版本、发布前终检、查看自己沉淀了什么）在对话里自然说出来就行——它听得懂人话。

## 安装

把下面这句发给 Cursor、Codex、Claude Code 或其他支持 Agent Skills 的工具：

```text
请安装这个 skill：https://github.com/yuwen-cool/yuwen-publish-precheck.git
```

或手动克隆到技能目录：

```bash
git clone https://github.com/yuwen-cool/yuwen-publish-precheck.git ~/.cursor/skills/yuwen-publish-precheck
```

零依赖（仅需 Python 3）。

### 升级（不会覆盖你的沉淀）

在 skill 目录里执行 `git pull` 即可。你的数据受三层保护：

1. 你后续新增的文件（词库、案例、审核记录）不在仓库追踪范围内（.gitignore），`git pull` 永远不会碰它们；
2. 随包发布的模板文件（profile.md、my-rules.md、expressions.md 等）**发布后上游冻结**，不再修改——你在里面写的内容不会遇到上游变更；
3. 即使极端情况下发生冲突，git 会**中止更新并提示**，绝不会静默覆盖你的本地修改。

升级只更新规则文档、脚本和说明文件。

## 覆盖范围（V1.0）

| 维度 | 覆盖 |
|---|---|
| 平台 | 抖音、小红书、微信视频号（快手在规划中，见 ROADMAP） |
| 内容 | 口播稿、文章、图文笔记文案、字幕、标题等文本（封面/画面检查清单在规划中） |
| 规则 | 通用安全 14 条 + 商业表达 15 条 + 医疗行业 7 条 + 金融行业 6 条，全部锚定官方来源 |
| 沉淀 | 个人词库、白名单、语义笔记、被罚案例复盘 |

## 边界（诚实声明)

- 不承诺 100% 过审——平台最终判定受内部模型、账号状态等不可见因素影响；
- 不提供任何绕审技巧；
- "可以发"指本次检查范围内未发现阻断问题；
- 完全本地运行，你的稿件和数据不上传任何第三方。
